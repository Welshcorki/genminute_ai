# Minute AI - 발표 PPT 개발 파트 (재구성)

> **배치**: "서비스 아키텍처 및 구현 방안" 섹션에 사용
> **슬라이드 수**: 3-4개 권장

---

## 📌 PPT 구성 제안

### 슬라이드 1: 기술 스택 (Technology Stack)
**목적**: 어떤 기술들을 사용했는지 한눈에 보여주기

### 슬라이드 2: 시스템 아키텍처 (System Architecture)
**목적**: 전체 시스템 구조를 시각적으로 표현

### 슬라이드 3: 서비스 플로우 차트 (Service Flow)
**목적**: 사용자 관점에서 서비스가 어떻게 작동하는지 보여주기

### 슬라이드 4 (선택): 핵심 기능 처리 흐름
**목적**: STT → 요약 → 챗봇의 기술적 처리 과정

### Frontend
```
🎨 UI Framework

```

**특징**:
- 단순하고 빠른 로딩

---

### Backend
```
⚙️ Web Framework
├─ Python 3.9+
├─ Flask 3.0
├─ RESTful API 설계
└─ Session 기반 인증
```

**특징**:
- 경량 프레임워크로 빠른 개발
- 마이크로서비스 아키텍처 가능

---

### AI/ML
```
🤖 AI Models & Services
├─ Google Gemini 2.5 Pro
│   ├─ STT (Speech-to-Text)
│   ├─ Speaker Diarization
│   └─ Text Summarization
│
├─ OpenAI Embeddings
│   └─ text-embedding-3-small (768차원)
│
└─ LangChain Framework
    ├─ Vector Store Integration
    ├─ Self-Query Retriever
    └─ RAG (Retrieval-Augmented Generation)
```

**특징**:
- **Gemini 2.5 Pro**: 2025년 최신 멀티모달 모델
  - 오디오 직접 입력 가능 (별도 전처리 불필요)
  - 화자 분리 자동 처리
  - 한국어 고성능 STT

- **OpenAI Embeddings**: 의미 기반 검색
  - 768차원 벡터로 텍스트 임베딩
  - 높은 정확도의 semantic search

- **LangChain**: AI 애플리케이션 프레임워크
  - Vector DB 연동 간소화
  - RAG 파이프라인 구축

---

### Database
```
💾 Data Storage
├─ SQLite 3.x (관계형 DB)
│   ├─ users (사용자 정보)
│   ├─ meeting_dialogues (STT 원본)
│   ├─ meeting_minutes (회의록)
│   └─ meeting_shares (공유 관계)
│
└─ ChromaDB (Vector DB)
    ├─ meeting_chunk (청크 단위 벡터)
    └─ meeting_subtopic (주제별 벡터)
```

**특징**:
- **SQLite**:
  - 서버리스 DB로 간단한 배포
  - 메타데이터 및 관계 데이터 저장

- **ChromaDB**:
  - Python 네이티브 Vector DB
  - 의미 검색 (Semantic Search)
  - 빠른 유사도 검색 (<100ms)

---

### Authentication

🔐 보안 & 인증

**특징**:
- Firebase로 소셜 로그인 간편 구현

- `@login_required` 데코레이터로 보안 강화

**특징**:

## 📱 상세 사용자 액션

### 단계별 사용자 경험

#### 1단계: 로그인
```
입력: Google 계정 클릭
처리: Firebase Authentication
출력: 메인 대시보드로 이동
```

#### 2단계: 파일 업로드
```
입력:
  ├─ 제목: "2025년 1월 팀 회의"
  ├─ 날짜: 2025-01-06
  └─ 파일: team_meeting.mp4 (107MB)

처리:
  ├─ 파일 형식 체크 (.mp4 → 허용)
  ├─ uploads/ 폴더에 저장
  └─ 비디오인 경우 ffmpeg로 WAV 변환

출력:
  └─ "업로드 완료! AI 처리 시작..." 메시지
```

#### 3단계: AI 처리 (백그라운드)
```
처리 1: STT (Gemini 2.5 Pro)
  ├─ 오디오 → 텍스트
  ├─ 화자 자동 분리
  └─ 타임스탬프 생성
  → meeting_dialogues 테이블 저장

처리 2: 요약 생성 (Gemini 2.5 Pro)
  ├─ 전체 대화 분석
  ├─ 주제별 그룹화
  └─ 구조화된 요약 생성
  → meeting_subtopic 저장

처리 3: 벡터화 (OpenAI Embeddings)
  ├─ 텍스트 → 768차원 벡터
  └─ 청크 단위 임베딩
  → ChromaDB 저장

소요 시간: 약 3-5분 (파일 크기에 따라)
```

#### 4단계: 회의록 확인
```
화면 구성:
  ├─ 좌측: 타임라인 뷰
  │   └─ [00:05] Speaker 0: "회의를 시작하겠습니다"
  │   └─ [00:12] Speaker 1: "네, 좋습니다"
  │
  ├─ 중앙: AI 생성 요약
  │   └─ ### 주요 안건
  │   └─ * 2025 Q1 목표 설정
  │   └─ * 마케팅 예산 논의
  │
  └─ 우측: 화자별 점유율
      └─ Speaker 0: 35%
      └─ Speaker 1: 52%
      └─ Speaker 2: 13%
```

#### 5단계: 챗봇 질의응답
```
사용자: "이번 회의의 핵심 결정사항은?"

AI 처리:
  1. 질문 벡터화
  2. ChromaDB에서 관련 청크 검색
  3. 검색된 context + 질문 → Gemini
  4. 답변 생성

응답:
  "주요 결정사항은 다음과 같습니다:
   1. Q1 매출 목표 10% 상향
   2. 마케팅 예산 20% 증액
   3. 신제품 출시 일정 3월로 확정

   출처: [00:25-01:30], [02:15-03:00]"
```

#### 6단계: 공유
```
입력:
  └─ 공유할 사용자 이메일: "team@company.com"

처리:
  ├─ 이메일로 사용자 검색
  ├─ meeting_shares 테이블에 공유 관계 저장
  └─ 피공유자에게 읽기 권한 부여

출력:
  └─ "team@company.com에게 공유되었습니다"
```
