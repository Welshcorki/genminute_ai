## 슬라이드 2: 문제 정의

### 📌 현재의 문제점
- 회의 후 회의록 작성에 **평균 30분~1시간** 소요
- 회의 중 필기로 인한 **집중도 저하**
- 회의록 작성자의 **주관적 해석 개입**
- 나중에 회의 내용 찾기 어려움

### 💡 해결하고자 하는 것
> "회의에 집중하고, AI가 회의록을 자동으로 생성하게 하자!"

---

## 슬라이드 3: 솔루션 개요

### Minute AI란?
**오디오/비디오 파일을 업로드하면 자동으로 회의록을 생성하는 웹 서비스**

### 핵심 기능
1. 🎤 **자동 음성인식** (STT)
2. 📝 **AI 회의록 요약**
3. 💬 **대화형 AI 챗봇**
4. 📊 **화자별 분석**
5. 👥 **팀원과 공유**

---

## 슬라이드 4: 시스템 아키텍처

```
[사용자] → [Flask Web App] → [Gemini 2.5 Pro API]
                ↓                      ↓
          [SQLite DB]            [STT + 요약]
                ↓
          [ChromaDB]
         (Vector DB)
                ↓
          [AI 챗봇]
```

### 기술 스택
**Backend**: Python, Flask
**AI/ML**: Google Gemini 2.5 Pro (STT, Summarization, Q&A)
**Database**: SQLite (메타데이터), ChromaDB (벡터 검색)
**Auth**: Firebase Authentication
**Frontend**: HTML/CSS/JavaScript, Jinja2

---

## 슬라이드 5: 주요 기능 - STT

### 🎤 자동 음성 인식 (Speech-to-Text)

**지원 포맷**: MP3, WAV, M4A, FLAC, MP4

**특징**:
- Gemini 2.5 Pro의 고성능 STT 엔진
- 화자 자동 분리 (Speaker Diarization)
- 타임스탬프 자동 생성
- 비디오 파일 자동 오디오 추출

**결과물**:
```
[Speaker 0, 00:05] 오늘 회의 시작하겠습니다.
[Speaker 1, 00:12] 네, Q1 목표에 대해 논의하죠.
```

---

## 슬라이드 6: 주요 기능 - 회의록 생성

### 📝 AI 회의록 자동 요약

**처리 과정**:
1. STT 완료된 텍스트 수신
2. Gemini 2.5 Pro로 요약 생성
3. 구조화된 회의록 저장

**생성 내용**:
- 회의 주제
- 핵심 내용 요약
- 결정 사항
- 액션 아이템
- 다음 단계

## 슬라이드 7: 주요 기능 - AI 챗봇

### 💬 회의 내용 기반 대화형 AI

**사용 예시**:
```
Q: "이번 회의의 핵심 결정사항은?"
A: 주요 결정사항은 다음과 같습니다:
   1. Q1 매출 목표 10% 상향
   2. 마케팅 예산 20% 증액
   3. ...

Q: "김과장님이 제안한 내용 정리해줘"
A: 김과장님은 다음을 제안하셨습니다:
   - 신규 프로모션 캠페인 기획
   - ...
```

**기술**:
- ChromaDB를 활용한 **Semantic Search** (의미 검색)
- RAG (Retrieval-Augmented Generation) 기반
- 출처(source) 제공으로 검증 가능

## 슬라이드 10: 사용자 시나리오

### 일반 사용 흐름
```
1. Google 계정으로 로그인
   ↓
2. 회의 녹음 파일 업로드
   ↓
3. AI가 자동으로 처리 (2-5분 소요)
   ↓
4. 회의록 확인 & 챗봇으로 질문
   ↓
5. 팀원에게 공유
```

### 예상 시간 절감
- **기존**: 1시간 회의 + 30분 회의록 작성 = **90분**
- **Minute AI**: 1시간 회의 + 5분 검토 = **65분**
- **절감**: **25분 (약 28%)**

## 슬라이드 18: 개발 과정 - 도전과제

### 🔥 주요 도전과제

1. **STT API 안정성**
   - 문제: Gemini API가 None 반환
   - 해결: 방어 코드 추가 및 에러 핸들링

2. **Vector DB 통합**
   - 문제: LangChain + ChromaDB 연동
   - 해결: 커스텀 래퍼 클래스 구현

3. **권한 관리 복잡도**
   - 문제: 소유자/공유자/어드민 구분
   - 해결: 데코레이터 패턴 활용

4. **삭제 데이터 정합성**
   - 문제: meeting_shares 테이블 누락
   - 해결: 삭제 로직 일관성 보장
