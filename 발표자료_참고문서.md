# 1. 프로젝트 소개 및 필요성

## 1.1 개요 및 필요성

### 프로젝트 개요
**GenMinute**는 회의 음성을 AI로 자동 분석하여 정확한 회의록을 생성하는 웹 기반 서비스입니다.

**핵심 가치 제안**:
- ⏱️ **시간 절약**: 회의 후 수기 작성에 드는 시간 90% 단축
- 🎯 **정확성**: AI 기반 화자 분리 및 맥락 파악으로 정확한 회의록 생성
- 🔍 **검색 가능**: 과거 회의 내용을 즉시 찾아볼 수 있는 AI 챗봇
- 👥 **협업 강화**: 팀원 간 회의록 공유 및 권한 관리

### 필요성

#### 현대 비즈니스 환경의 변화
- **회의 증가**: 원격 근무 확산으로 온라인 회의 300% 증가 (2020-2024)
- **정보 과부하**: 주당 평균 5-7회 회의 참여, 핵심 내용 기억 어려움
- **문서화 부담**: 회의록 작성에 평균 30-60분 소요

#### 대상 사용자
- 📊 **기업 팀**: 프로젝트 회의, 주간 회의, 브레인스토밍
- 🏫 **교육 기관**: 강의 녹음, 세미나, 연구 미팅
- 🏛️ **공공 기관**: 회의 기록 의무화 대응
- 💼 **프리랜서**: 클라이언트 미팅 기록

# 2. 문제 정의 및 AI 기반 솔루션

## 2.1 수기 회의록 문제점

### 기존 회의록 작성의 Pain Points

#### 1️⃣ 시간 낭비
```
회의 시간: 60분
회의록 작성 시간: 30-60분
총 소요 시간: 90-120분
→ 실제 업무 시간 감소
```

#### 2️⃣ 정보 누락 및 왜곡
- **기억 의존**: 회의 중 모든 내용 기억 불가능
- **주관 개입**: 작성자의 해석으로 원래 의도 왜곡
- **일관성 부족**: 작성자마다 다른 형식과 수준

#### 3️⃣ 검색 및 재활용 어려움
- **PDF/Word 문서**: Ctrl+F 검색만 가능
- **맥락 파악 불가**: "그때 그 프로젝트 일정이 뭐였지?" → 모든 회의록 뒤지기
- **인사이트 발굴 불가**: 과거 회의에서 언급된 아이디어 묻힘

#### 4️⃣ 접근성 문제
- **회의 불참자**: 회의록만으로 충분한 맥락 이해 어려움
- **다국어 팀**: 번역 필요
- **청각 장애인**: 음성 정보 접근 불가

## 2.2 핵심 기능 및 차별화 전략

### GenMinute의 핵심 기능

#### 🎤 1. AI 기반 STT (Speech-to-Text)
**기술**: Gemini 2.5 Pro (멀티모달)

**기능**:
- 음성 → 텍스트 변환 (정확도 95%+)
- 자동 화자 분리 (최대 10명)
- 타임스탬프 자동 기록

**차별점**:
- 한국어 자연어 처리 최적화
- 회의 특화 용어 인식 (프로젝트명, 전문 용어 등)

---

#### 📝 2. Smart Chunking (지능형 청킹)
**기술**: 화자/시간 기반 의미적 분할

**동작 방식**:
```
[기존 STT]
SPEAKER_00: 안녕하세요 오늘 회의는...
SPEAKER_01: 네 저는...
SPEAKER_00: 그리고 이번에...

↓ Smart Chunking

[Chunk 1] 120-180초, 화자 2명
"회의 시작 및 안건 소개
 SPEAKER_00이 프로젝트 배경 설명
 SPEAKER_01이 진행 상황 공유"
```

**효과**:
- 문맥 단위로 분할하여 RAG 검색 정확도 향상
- 의미 없는 단어 단위 분할 방지

---

#### 🤖 3. AI 요약 및 회의록 생성
**기술**: Gemini 2.5 Pro + RAG

**2단계 요약 시스템**:

**1단계: 소주제별 요약**
```markdown
### 프로젝트 일정 및 마일스톤
* 1차 마일스톤: 2월 1일 프로토타입 완성 [cite: 1,3]
* 최종 완료: 2월 15일 배포 [cite: 5]
* 리스크: 디자인 리소스 부족 우려 [cite: 7]
```

**2단계: 구조화된 회의록**
- 회의 제목 및 일시
- 참석자 목록
- 주요 안건
- 결정 사항
- 액션 아이템 (담당자, 마감일)

**차별점**:
- `[cite: N]` 인용 표시로 출처 추적 가능
- RAG로 원본 대화 기반 정확성 보장

---

#### 💬 4. AI 챗봇 (Self-Query Retriever)
**기술**: Gemini 2.5 Flash + ChromaDB

**기능**:
```
사용자: "지난달 마케팅 예산 얼마로 결정됐어?"

챗봇: "지난 11월 15일 회의에서 마케팅 예산을
       500만원으로 결정했습니다. [출처: 2024-11-15 회의록]"
```

**검색 방식**:
- **Chunk 검색**: 실제 대화 원문 3개
- **Subtopic 검색**: 요약본 3개
- **병합 답변**: 6개 문서 기반 종합 답변

**보안**:
- 본인이 작성한 노트 + 공유받은 노트만 검색
- 권한 없는 노트는 절대 접근 불가

---

#### 👥 5. 협업 기능
**사용자 권한 관리**:
- **Owner**: 노트 생성자, 수정/삭제/공유 권한
- **Shared User**: 조회 및 챗봇 사용 가능
- **Admin**: 모든 노트 접근 (디버깅용)

**공유 기능**:
- 이메일 기반 노트 공유
- 공유받은 노트 별도 페이지에서 확인
- 공유 해제 가능

---

#### 🎬 6. 멀티미디어 지원
**지원 포맷**:
- **오디오**: WAV, M4A, MP3
- **비디오**: MP4 (자동 오디오 추출)

**변환 프로세스**:
```
MP4 업로드
  ↓ ffmpeg 자동 변환
WAV 파일 생성 (16kHz, 모노)
  ↓ Gemini STT
텍스트 변환
  ↓ 임시 파일 자동 삭제
완료
```

### Database

#### SQLite (관계형 DB)
**테이블 구조**:
```sql
users (사용자 정보)
  ├─ id, google_id, email, name, role

meeting_dialogues (대화 원본)
  ├─ meeting_id, speaker_label, start_time, segment
  └─ owner_id (FK → users.id)

meeting_minutes (회의록)
  ├─ meeting_id, title, minutes_content
  └─ owner_id (FK → users.id)

meeting_shares (공유 관계)
  ├─ meeting_id, owner_id, shared_with_id
  └─ shared_at
```

#### ChromaDB (벡터 DB)
**컬렉션**:
```
meeting_chunk (대화 청크)
  - Embedding: OpenAI text-embedding-3-small
  - 메타데이터: meeting_id, start_time, end_time, speaker_count

meeting_subtopic (요약본)
  - Embedding: OpenAI text-embedding-3-small
  - 메타데이터: meeting_id, main_topic
```

## 3.3 서비스 플로우 차트 (소비자 관점)

### 전체 사용자 여정

```
1. 로그인
   │
   ├─ Google 계정으로 로그인
   └─ Firebase 인증

2. 회의 파일 업로드
   │
   ├─ 파일 선택 (MP4, WAV, M4A)
   ├─ 제목 입력
   └─ 업로드 (프로그레스바)

3. AI 자동 처리 (30-90초)
   │
   ├─ STT (음성 → 텍스트)
   ├─ 화자 분리
   ├─ Smart Chunking
   └─ 소주제별 요약 생성

4. 회의록 확인
   │
   ├─ 대화록 탭: 타임스탬프별 대화 내용
   ├─ 요약 탭: 소주제별 요약
   └─ 회의록 탭: 구조화된 최종 회의록

5. AI 챗봇 활용
   │
   ├─ 자연어 질문 입력
   ├─ AI 검색 (본인 노트 + 공유받은 노트)
   └─ 출처 포함 답변 확인

6. 협업 (선택)
   │
   ├─ 팀원에게 노트 공유 (이메일)
   └─ 공유받은 노트 조회
```

### 상세 플로우차트

#### 📤 업로드 플로우
```
[사용자] 파일 선택
    ↓
[시스템] 파일 유효성 검사 (크기, 형식)
    ↓
[브라우저] 프로그레스바 표시
    ↓
[서버] 파일 저장 (/uploads)
    ├─ MP4? → ffmpeg 변환 (WAV)
    └─ WAV? → 그대로 사용
    ↓
[Gemini] STT 처리 (30-60초)
    ├─ 음성 → 텍스트
    ├─ 화자 분리 (SPEAKER_00, 01...)
    └─ 타임스탬프 기록
    ↓
[SQLite] meeting_dialogues 저장
    ↓
[Smart Chunking] 의미 단위 분할
    ↓
[ChromaDB] meeting_chunk 저장
    ↓
[Gemini] 소주제별 요약 생성
    ↓
[ChromaDB] meeting_subtopic 저장
    ↓
[브라우저] /view/{meeting_id} 리다이렉트
```

### 기능 2: Smart Chunking

#### 알고리즘
```python
def _create_smart_chunks(segments):
    """
    화자 변경 또는 30초 경과 시 청크 분할
    """
    chunks = []
    current_chunk = []
    prev_speaker = None
    chunk_start_time = 0

    for seg in segments:
        # 조건 1: 화자 변경
        if seg['speaker'] != prev_speaker:
            if current_chunk:
                chunks.append(merge(current_chunk))
            current_chunk = [seg]
            prev_speaker = seg['speaker']
            chunk_start_time = seg['start_time']

        # 조건 2: 30초 경과
        elif seg['start_time'] - chunk_start_time > 30:
            chunks.append(merge(current_chunk))
            current_chunk = [seg]
            chunk_start_time = seg['start_time']

        else:
            current_chunk.append(seg)

    return chunks
```

| 시간 단위 (10초) | 180개 | 75% |

## 5.5 Q&A 예상 질문

### 기술 관련

**Q1. Whisper API 대신 Gemini를 선택한 이유는?**
```
A. Gemini 2.5 Pro는 멀티모달 입력을 지원하여
   음성 파일을 직접 처리할 수 있으며,
   한국어 자연어 처리 성능이 우수합니다.
   또한 동일한 모델로 STT + 요약을 통합 처리하여
   아키텍처가 단순해집니다.
```

**Q2. 개인정보 보호는 어떻게 하나요?**
```
A. 1) 사용자 권한 관리: 본인 노트 + 공유받은 노트만 접근
   2) Firebase Authentication: 구글 표준 보안
   3) 온프레미스 옵션: 향후 제공 예정 (기업용)
   4) 데이터 암호화: 전송 중 HTTPS, 저장 시 암호화
```

**Q3. 실시간 STT는 언제 지원되나요?**
```
A. 현재는 파일 업로드 방식만 지원합니다.
   실시간 STT는 WebSocket 기반으로
   2-3개월 내 베타 버전 출시 예정입니다.
```
