### 2. Critical Bug 수정: 삭제 시 meeting_shares 미삭제

#### 문제점
노트 삭제 시 `meeting_shares` 테이블의 공유 관계 레코드가 삭제되지 않아 데이터 불일치 발생

#### 수정 내역

**파일 1**: `utils/db_manager.py`
- `delete_meeting_by_id()` 함수 수정 (204-311줄)
- meeting_shares 삭제 전 개수 확인 추가 (238-246줄)
- meeting_shares 삭제 수행 추가 (261-266줄)
- meeting_shares 삭제 후 검증 추가 (288-293줄)
- 반환값에 shares 정보 추가 (308줄)

**파일 2**: `utils/vector_db_manager.py`
- `_delete_all_meeting_data()` 함수 최종 요약 수정 (745-768줄)
- SQLite meeting_shares 삭제 로그 추가 (751줄)
- 반환값에 sqlite_shares 정보 추가 (763줄)

#### 영향
- 노트 삭제 시 공유 관계도 함께 삭제되어 데이터 정합성 보장
- 삭제 로그에 meeting_shares 정보 표시

### 예상 질문 & 답변 준비
1. **Q: 왜 Gemini를 선택했나요?**
   - A: 최신 모델(2.5 Pro), 높은 STT 정확도, 한국어 지원 우수

2. **Q: 데이터 보안은 어떻게 하나요?**
   - A: Firebase 인증, 소유자 기반 권한 관리, 세션 암호화

3. **Q: 실시간 회의 녹음은 가능한가요?**
   - A: 현재는 파일 업로드만 지원, 향후 실시간 스트리밍 STT 추가 예정

4. **Q: 비용은 얼마나 드나요?**
   - A: Gemini API 종량제 (1시간 회의 약 $0.5-1), 프리미엄 모델 고려 중

5. **Q: 다른 언어도 지원하나요?**
   - A: Gemini는 100+ 언어 지원, 현재는 한국어 최적화, 영어/일본어 추가 예정
