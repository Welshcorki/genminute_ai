# 📹 Demo Filming Guide
## Minute AI 데모 영상 촬영 가이드

---

## 1. 기본 구조 (3-5분 권장)

### A. 인트로 (15초)
- "Minute AI - AI 기반 회의록 자동 생성 시스템"
- 문제 제기: "회의록 작성에 시간 낭비하고 계신가요?"

### B. 메인 시나리오 (3-4분)
```
1. 로그인 → 2. 업로드 → 3. 결과 확인 → 4. 챗봇 → 5. 공유
```

### C. 아웃트로 (15초)
- 주요 기능 요약
- 기술 스택 언급 (Gemini 2.5 Pro, ChromaDB 등)

---

## 2. 상세 시나리오

### **Scene 1: 로그인 & 대시보드 (20초)**
- Google 로그인 버튼 클릭
- 로그인 후 왼쪽 메뉴 간략히 보여주기
- "새노트 만들기" 메뉴 강조

### **Scene 2: 회의 파일 업로드 (30초)**
- 미리 준비한 샘플 오디오/비디오 파일 선택
  - **추천**: 2-3분 분량의 실제 회의 녹음 (한국어)
  - 예: "2025년 Q1 마케팅 전략 회의"
- 제목 입력: "Q1 마케팅 전략 회의"
- 날짜 선택: 2025-01-15
- "업로드 및 처리 시작" 버튼 클릭
- 로딩 화면 (빨리감기 효과 사용)

### **Scene 3: 회의록 결과 확인 (60초)**
- 자동 생성된 회의록 화면 등장
- **중요 요소들 클로즈업:**
  1. **타임라인 뷰**: 화자별 타임스탬프와 대화 내용
  2. **요약 섹션**: AI가 생성한 회의 요약
  3. **비디오 플레이어**: 특정 구간 클릭 → 해당 시간으로 이동
  4. **화자별 점유율 차트**: 각 참석자의 발언 비율 시각화

### **Scene 4: AI 챗봇 활용 (45초)**
- 오른쪽 챗봇 아이콘 클릭
- **실용적인 질문 시연:**
  ```
  Q1: "이번 회의의 핵심 결정사항은 뭐야?"
  → AI 응답 표시

  Q2: "김과장님이 제안한 내용 정리해줘"
  → AI 응답 표시

  Q3: "다음 액션 아이템은?"
  → AI 응답 표시
  ```

### **Scene 5: 노트 관리 & 공유 (40초)**
- "노트 보기" 메뉴 클릭 → 저장된 노트 목록
- 노트 체크박스 선택
- **"공유" 버튼 클릭 (신규 기능!)**
  - 이메일 입력: "colleague@company.com"
  - 공유 완료 메시지
- "공유 노트 보기" 메뉴로 이동
  - 다른 사용자가 공유한 노트 표시

### **Scene 6: 추가 기능 (선택, 20초)**
- 여러 노트 선택 후 일괄 삭제
- 검색 기능 (만약 있다면)

## 3. 촬영 팁

### **사전 준비**

#### 1. 테스트 계정 2개 준비
- 계정 A: 메인 시연용
- 계정 B: 공유 기능 시연용

#### 2. 샘플 데이터 준비
- 2-3분 분량의 회의 녹음 파일 (명확한 음성)
- 이미 생성된 노트 2-3개 (노트 목록이 풍성하게 보이도록)

#### 3. 브라우저 설정
- 불필요한 탭/북마크 정리
- 전체 화면 모드 또는 깔끔한 화면 구성
- 브라우저 개발자 도구 닫기

### **촬영 기법**

#### 1. 화면 녹화 도구
- **Windows**: OBS Studio, ShareX
- **Mac**: QuickTime, ScreenFlow
- **해상도**: 1920x1080 권장

#### 2. 자막/주석 추가
- 각 단계마다 간단한 텍스트 설명
- 예: "Gemini 2.5 Pro를 활용한 STT", "ChromaDB 기반 의미 검색"

#### 3. 속도 조절
- 업로드/처리 대기 시간: 빨리감기 (2-4배속)
- 중요 기능 시연: 정상 속도
- 반복 작업: 컷 편집

#### 4. 배경 음악 (선택)
- 은은한 BGM 추가 (저작권 프리)
- 나레이션 추가 시 음악 볼륨 낮춤

## 4. 시나리오별 버전

### **Version A: 일반 사용자용 (3분)**
```
문제 제기 → 업로드 → 결과 확인 → 챗봇 → 공유
```
- 기술 용어 최소화
- 사용 편의성 강조

### **Version B: 기술 시연용 (5분)**
```
아키텍처 소개 → 전체 플로우 → 각 기능 상세 → 어드민 기능
```
- Gemini API, ChromaDB, Vector Search 언급
- 디버그 메뉴 (스크립트 입력, 리트리버) 시연
- 콘솔 로그 보여주기

### **Version C: 비즈니스 피치용 (2분)**
```
문제 → 솔루션 → 핵심 가치 → Call-to-Action
```
- ROI 강조: "회의록 작성 시간 90% 절감"
- 차별점: "실시간 질문 응답 가능한 AI 챗봇"

## 5. 촬영 체크리스트

### **촬영 전**
- [ ] 테스트 계정 2개 로그인 확인
- [ ] 샘플 오디오 파일 준비 (2-3분, 명확한 음성)
- [ ] 기존 노트 2-3개 생성해두기
- [ ] 브라우저 화면 정리
- [ ] ngrok 터널 실행 (외부 접속용)
- [ ] 화면 녹화 도구 테스트

### **촬영 시**
- [ ] 마우스 움직임 부드럽게
- [ ] 중요 기능은 2-3초 정지 후 클릭
- [ ] 로딩 시간은 빨리감기 편집 예정

### **편집 시**
- [ ] 불필요한 대기 시간 컷
- [ ] 각 섹션마다 텍스트 자막 추가
- [ ] 인트로/아웃트로 추가
- [ ] 배경 음악 (선택)

## 7. 주요 강조 포인트

### 핵심 차별화 요소
1. **Gemini 2.5 Pro**: 최신 AI 모델 사용
2. **실시간 챗봇**: 회의 내용에 대해 질문 가능
3. **화자별 분석**: 참석자별 발언 비율 시각화
4. **공유 기능**: 팀 협업 지원
5. **비디오 연동**: 클릭만으로 특정 구간 이동

### 기술 스택 (기술 시연용)
- **Frontend**: Flask, Jinja2, JavaScript
- **AI/ML**: Gemini 2.5 Pro (STT, Summarization)
- **Vector DB**: ChromaDB (Semantic Search)
- **Database**: SQLite
- **Auth**: Firebase Authentication
